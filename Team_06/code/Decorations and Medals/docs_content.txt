============================================================
DECORATIONS AND MEDALS DATA
============================================================

=== PARAGRAPHS ===
0: Decorations and Medals Process Data
1: The data can be grouped into two main categories: data collected at the start, and data retrieved for routing and process task approval.
2: 1. Application Data (Initiation)
3: This data is collected at the very beginning of the process, originating from the "Submit application for distinction" task, and forms the core of the request.
4: Employee Name (employee_name: String): The full name of the nominated employee.
5: Organizational Unit (organizational_unit: String): The organizational unit of the nominated employee.
6: Decoration Type (decoration_type: String): The specific state or resort-level medal/decoration being requested.
7: Application Justification (application_justification: Text): A text field containing the reasons and achievements justifying the award.
8: 2. Process Approval and Routing Data
9: This data is generated during the process to track opinions and decisions and to route the flow correctly.
10: Reviewer Opinion (reviewer_opinion: String): A text variable to store the opinion from the "Review applications and forward to PD" task, completed by the PRK / Chancellor.
11: RKR Decision (rkr_decision: List [ĹAcceptedĺ, ĹRejectedĺ]): The variable set during the "Make decision" task by the Rector. This variable is used by the gateway "Application accepted?" to determine the next step. (e.g., ĹAcceptedĺ or ĹRejectedĺ).
12: Award Grant Date (award_grant_date: Date): The date the award was officially granted, which would be entered during the "Enter decoration into register" task.
13: Process Outcome (process_outcome: List [ĹAcceptedĺ, ĹRejectedĺ]): A final status variable for logging (e.g., ĹCompletedĺ, ĹRejectedĺ).
